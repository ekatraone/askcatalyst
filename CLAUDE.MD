## RAG Bot Architecture (Azure-Based)

### Architecture Overview
```
User Query → API/Azure Function → Assistant Manager → Azure OpenAI Assistants API
                                         ↓
                                  Vector Store Manager → Azure OpenAI Vector Store
                                         ↓
                                  Database Manager → Cosmos DB
                                         ↓
                                  Response with Citations
```

### Core Components

#### 1. **Assistant Manager** (`assistant_manager.py`)
- Manages Azure OpenAI Assistants API integration
- Handles conversation threads and context
- Processes user messages with RAG capabilities
- Extracts citations from file sources
- Tracks processing time and metadata

#### 2. **Vector Store Manager** (`vector_store_manager.py`)
- Handles document uploads to Azure OpenAI
- Manages vector store operations (create, list, delete)
- Validates file formats and sizes (max 512MB)
- Supports: PDF, TXT, MD, JSON, DOCX, HTML, CSV, XLSX
- Auto-loads documents from `documents/` directory
- Attaches vector store to assistants

#### 3. **Database Manager** (`database.py`)
- Cosmos DB integration for persistent storage
- User profiles and interaction tracking
- Conversation history logging
- Analytics and metrics collection
- GDPR-compliant data deletion

#### 4. **API Layer** (`api.py` & `function_app.py`)
- **Flask API**: Local development and testing
- **Azure Functions**: Production serverless deployment
- REST endpoints for queries, uploads, and analytics
- Health checks and service monitoring

### Technology Stack
- **Cloud Platform**: Microsoft Azure
- **AI Service**: Azure OpenAI (GPT-4o with Assistants API)
- **Vector Store**: Azure OpenAI native vector store
- **Database**: Azure Cosmos DB (NoSQL)
- **Compute**: Azure Functions (serverless)
- **Framework**: Python 3.8+
- **Web Frameworks**: Flask (dev) + Azure Functions (prod)
- **Document Processing**: PyPDF2, python-docx, beautifulsoup4

## Setup Instructions

### Prerequisites
```bash
python >= 3.8
pip
MongoDB instance
```

### Installation
```bash
# Clone the repository
git clone <repository-url>
cd askcatalyst

# Checkout development branch
git checkout claude/setup-ragbot-ask-catalyst-i4WeR

# Install dependencies
pip install -r requirements.txt
```

### Environment Variables
Create a `.env` file with the following variables:

```env
# Azure OpenAI Configuration
AZURE_OPENAI_API_KEY=<your-azure-openai-api-key>
AZURE_OPENAI_ENDPOINT=https://<your-resource-name>.openai.azure.com/
AZURE_OPENAI_API_VERSION=2024-05-01-preview
AZURE_OPENAI_DEPLOYMENT_NAME=gpt-4o
AZURE_OPENAI_ASSISTANT_ID=<auto-generated-on-first-run>
AZURE_OPENAI_VECTOR_STORE_ID=<auto-generated-on-first-run>

# Cosmos DB Configuration
COSMOS_DB_ENDPOINT=https://<your-account>.documents.azure.com:443/
COSMOS_DB_KEY=<your-cosmos-db-key>
COSMOS_DB_DATABASE=askcatalyst

# Legacy Configuration (for existing modules)
MONGODB_URI=<your-mongodb-connection-string>
WATI_API_KEY=<your-wati-key>

# Server Configuration
PORT=5000
FLASK_DEBUG=False
```

**Note**: The `AZURE_OPENAI_ASSISTANT_ID` and `AZURE_OPENAI_VECTOR_STORE_ID` will be auto-generated on first run. Check the logs for the generated IDs and add them to your `.env` file.

## RAG Bot API Endpoints

### Query Processing
**POST** `/api/query`
```json
{
  "user_id": "user123",
  "message": "What is RAG?",
  "thread_id": "thread_abc"  // optional
}
```
Response includes answer, citations, and thread_id for context continuity.

### Document Management
**POST** `/api/upload` - Upload documents to vector store
```json
{
  "file_paths": ["/path/to/doc1.pdf", "/path/to/doc2.txt"]
}
```

**GET** `/api/vector-store/info` - Get vector store information

**GET** `/api/vector-store/files` - List all files

**DELETE** `/api/vector-store/files/<file_id>` - Delete a file

### User Management
**GET** `/api/user/<user_id>` - Get user profile

**GET** `/api/history/<user_id>?limit=10` - Get conversation history

### Analytics
**GET** `/api/analytics?date=2024-01-15` - Get analytics data

### Health Check
**GET** `/health` - Service health and component status

## Deployment

### Local Development
```bash
cd rag-bot
python api.py
```
The Flask API will start on port 5000 (or PORT env variable).

### Azure Functions Deployment
```bash
# Install Azure Functions Core Tools
npm install -g azure-functions-core-tools@4

# Navigate to rag-bot directory
cd rag-bot

# Deploy to Azure
func azure functionapp publish <your-function-app-name>
```

### Configuration in Azure
1. Set up Azure OpenAI resource
2. Create Cosmos DB account
3. Configure Function App settings with environment variables
4. Enable system-assigned managed identity (optional)
5. Set up monitoring with Application Insights

### Docker Deployment (Optional)
```bash
docker build -t askcatalyst-rag-bot .
docker run -p 5000:5000 --env-file .env askcatalyst-rag-bot
```
